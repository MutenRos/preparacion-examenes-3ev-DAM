Un correo electrónico:
1.-Recibe (IMAP o POP)
2.-Envía (SMTP)

Cuando abro la aplicación:
1.-Me conecto al servidor
2.-Miro cuantos mensajes hay
3.-Me los descargo
(qué ocurre si hay muchos mensajes en el servidor? me los descargo cada vez)
4.-Necesito algun tipo de almacenamiento de datos
5.-Cuando me conecte al servidor, solo quiero descargar los correos nuevos - para ir más rápido

MySQL - posible pero lo veo un poco grande
SQLite - mas pequeño, adecuado para algo personal
Archivo JSON - donde puedo guardar un correo en cada JSON - estructurado, nuevo - no está en motor
Archivo XML - igual que JSON, pero basado en XML - estructurado - vieji
Archivo EML - Formato específico de correo electrónico - estandar - viejo
Base de datos MongoDB - Ventajas JSON + busquedas - estructurado, busquedas - VPS
