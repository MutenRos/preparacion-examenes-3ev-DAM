// python vs c — 2.000.000.000 iteraciones vacías

Dos programas que hacen exactamente lo mismo: un bucle vacío de dos mil millones de iteraciones. Uno en Python interpretado, otro compilado a código máquina en C. La diferencia en tiempo es la diferencia en energía consumida. No hay matplotlib, no hay RAPL — solo time.time() y clock().

// benchmark — tiempo de ejecución típico

Python
~90s
C
~3s

// factor ×20-50 más rápido en C → proporcional al consumo energético

Python 001-bucle en python.py
import time

inicio = time.time()
for i in range(2_000_000_000):
    pass
fin = time.time()

print(f"Tiempo: {fin - inicio:.2f}s")
C 002-bucle en c.c
#include <stdio.h>
#include <time.h>

int main() {
    clock_t i = clock();
    for (long long j = 0;
         j < 2000000000LL; j++);
    clock_t f = clock();
    printf("Tiempo: %f s
",
      (double)(f-i)/CLOCKS_PER_SEC);
}
