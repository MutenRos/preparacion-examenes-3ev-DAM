🖥️ 1. Tipos de servidor

Fundamentalmente tenemos dos tipos:

Apache (puerto 80)Flask (puertos altos)
Multipropósito. Te abre /var/www/html y dentro pones lo que quieras en subcarpetas. Stack LAMP (PHP). Trabaja en un puerto determinado: 5000, 5001, 5002… Pero la gente solo carga por el puerto 80 → necesitamos proxy inverso.

🌐 2. Virtualhost de jocarsa

Cada dominio o subdominio lleva a su propia carpeta:

<VirtualHost *:80>
    ServerName jocarsa.com
    ServerAlias www.jocarsa.com
    DocumentRoot /var/www/html/jocarsa
    <Directory /var/www/html/jocarsa>
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>
sudo a2ensite jocarsa.com.conf
sudo apachectl configtest
sudo systemctl reload apache2

🎨 3. Subdominio colores.jocarsa.com

Mismo patrón, distinta carpeta:

<VirtualHost *:80>
    ServerName colores.jocarsa.com
    DocumentRoot /var/www/html/colores
</VirtualHost>
📌 El DNS debe apuntar el subdominio a la IP del servidor (registro A o CNAME).

🔁 4. Configurar Apache como proxy inverso

El usuario entra por el puerto 80, Apache redirige internamente a Flask en 127.0.0.1:5000.

🌐 Usuario :443
→
🛡️ Apache vhost
→
⚙️ gunicorn 127.0.0.1:5000
→
🐍 Flask app
sudo a2enmod proxy proxy_http headers
sudo systemctl restart apache2
<VirtualHost *:443>
    ServerName flask.jocarsa.com
    SSLEngine on
    SSLCertificateFile /etc/apache2/ssl/jocarsa_combined.cer
    SSLCertificateKeyFile /etc/apache2/ssl/jocarsa.key

    ProxyPreserveHost On
    RequestHeader set X-Forwarded-Proto "https"
    ProxyPass / http://127.0.0.1:5000/
    ProxyPassReverse / http://127.0.0.1:5000/
</VirtualHost>

⚙️ 5. Flask como servicio (systemd + gunicorn)

Para que la app arranque sola y se reinicie si cae:

pip install gunicorn

app.py con ProxyFix:

from flask import Flask
from werkzeug.middleware.proxy_fix import ProxyFix

app = Flask(__name__)
app.wsgi_app = ProxyFix(app.wsgi_app, x_for=1, x_proto=1, x_host=1)

@app.route("/")
def home():
    return "Hello from Flask with SSL"

/etc/systemd/system/flask-jocarsa.service:

[Unit]
Description=Flask app for flask.jocarsa.com
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/home/miniservidor/
Environment="PATH=/home/miniservidor/venv/bin"
ExecStart=/home/miniservidor/venv/bin/gunicorn --workers 2 --bind 127.0.0.1:5000 app:app
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable flask-jocarsa
sudo systemctl start flask-jocarsa
✅ Con esto Flask arranca al iniciar el servidor y sobrevive a reinicios.

📋 7. Resumen de contrataciones

Para tener una web profesional necesitas:

  • 🖥️ Servidor (VPS o dedicado)
  • 🌍 Dominio (registrado)
  • 🔒 Certificado SSL (Let's Encrypt es gratis)
  • 📧 Email asociado al dominio
  • 📊 Sistema de monitorización

📈 8. Comprobación de recursos

Comandos útiles para monitorizar el servidor:

htop                  # CPU + RAM en tiempo real
df -h                 # Espacio en disco
free -h               # Memoria
systemctl status XXX  # Estado de un servicio
journalctl -u XXX -f  # Logs en directo
