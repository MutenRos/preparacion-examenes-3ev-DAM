Repaso del despliegue

De nuestro equipo de desarrollo al servidor remoto: dos mundos que conviven a diario.

🖥️ Dos entornos, dos realidades

Nosotros desarrollamos en nuestro equipo local, pero la aplicación tiene que estar alojada en un servidor remoto. Son dos mundos muy distintos:

💻 Equipo de desarrollo

  • Local
  • OS con ventanas
  • Muy potente
  • Muy cómodo
  • Está en nuestra casa

🖥️ Servidor de producción

  • Remoto
  • OS de consola (Ubuntu Server)
  • Poco potente
  • No cómodo de usar
  • No está en nuestra casa

🚀 Dos formas de hacer deploy

📁 FTP (Filezilla)

Conectar y transferir archivos. Fácil, directo, sin versionado. Para proyectitos.

🔧 Git/GitHub recomendado

Más complejo pero mejor. Yo hago push a GitHub, el servidor hace pull. Ideal para proyectos grandes con versionado.

💻 Dev local
git push →
🐙 GitHub
git pull →
🖥️ Servidor

📋 Lo que necesitamos

  • Un servidor contratado (puede ser una máquina virtual)
  • Un dominio
  • Un certificado SSL
📌 Recordatorio Apache: la carpeta de publicación es /var/www/html/XXXXX
